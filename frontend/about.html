About Snitch

Platform information and release history

Current Release: v1.5
Snitch
AppSec Platform

An open-source application security platform that aggregates findings from multiple scanners, calculates per-application risk scores, enforces security policies in CI/CD, and delivers AI-powered remediation guidance — all from a single dashboard.

Release Notes

Major changes and features by version.

v1.5 Latest Feature

Checkov (IaC) & Grype (Container) Scanner Integration

  • Checkov scanner for Terraform, CloudFormation, ARM, and Bicep IaC files
  • Grype container image vulnerability scanning (set container_image on an application)
  • CI/CD push support for Checkov and Grype JSON output via /api/v1/cicd-scans/push
  • Policy engine extended to evaluate iac and container scan types
  • Deduplication keys updated for IaC (checkov_id + file) and container findings (cve + package + version)
v1.4 Feature

UI Overhaul — Sidebar Sections, Profile Widget, Settings, List View

  • Sidebar restructured into Overview / Config / Admin / Developer sections
  • Header user avatar widget with dropdown (profile link, logout placeholder)
  • Settings page for platform integration tokens (GitHub, Anthropic)
  • Applications page switched to sortable list/table view (default: risk score desc)
  • User profile page with light/dark mode toggle (persisted in localStorage)
v1.3 Security

Security Fixes — Cleartext Token Storage & ReDoS

  • Fixed CodeQL finding: GitHub tokens no longer stored in cleartext in localStorage (only presence flags)
  • Fixed ReDoS vulnerability in secrets pattern validation regex
v1.2 Feature

Shared Sidebar Component

  • Extracted sidebar to sidebar.js shared component — eliminates cross-page style inconsistencies
  • Active nav link detection based on current path with prefix matching
  • All HTML pages migrated to use <div id="sidebar-mount"> pattern
v1.1 Feature

Secrets Scanning with Gitleaks

  • Gitleaks integration for detecting leaked credentials in source code
  • Custom SecretPattern management via Secrets page and API
  • Secret masking — raw values never stored, only last 4 chars retained (****{last4})
  • Deduplication for secrets findings by rule ID and file path
v1.0 Feature

CI/CD Direct Push Ingestion

  • CI/CD scan push endpoint (POST /api/v1/cicd-scans/push) for ingesting scanner JSON directly from GitHub Actions
  • Automatic format detection (Semgrep, Grype, Checkov) in cicd_normaliser
  • Celery Beat periodic scanning tasks
v0.9 Feature

Rules Library & Policy Engine

  • Static catalog of ~37 rules across Semgrep, Checkov, and Gitleaks
  • Policy model and evaluator — define rules that auto-run after every scan
  • Policy actions: inform (log) or block (gates CI/CD deployments)
  • GitHub Repositories page for code scanning alert sync and fix PR creation
v0.8 Feature

User Profile & Light Mode

  • User profile page with avatar, display name, and role
  • Light / dark theme toggle with FOUC prevention via inline script
  • CSS design tokens in theme.css — severity palette, gradient, glow variables
  • Vibrant dark theme as default with full light mode support across all pages
